Подключение к сообществу

Для работы модуля необходимо создать сообщество и получить его ключ доступа.
Для этого переходим по ссылке и создаём сообщество.
Называем как угодно, выбираем тематику, например "Дом и дача", нажимаем "Создать сообщество". Далее можно выбрать аватарку и сразу добавить участников. Можно сделать это и позже.
Далее заходим в Управление и меняем Тип сообщества на "Частное" (или другое по желанию). Здесь же можно поменять аватарку и имя сообщества.
Далее в меню Сообщества выбираем пункт "Сообщения" и подпункт "Настройки для бота". Включаем "Возможности бота" и ставим галку "Добавить кнопку «Начать»"
После в меню Сообщества выбираем пункт "Дополнительно" и подпункт "Работа с API". Нажимаем на кнопку "Создать ключ" и проставляем все галочки. добавляя тем самым максимальные права. Нажимаем "Создать", подтверждаем кодом из СМС.
После этого копируем ключ и переходим в настройки модуля.

Регистрация и настройка пользователя

Регистрация
Добавляем подписчиков через меню Управленя сообществом (Подписчики -> Приглашения). Далее каждому подписчику для возможности принятия сообщений нужно нажать в клиенте кнопку "Начать".
Либо отправить любое сообщение.

Режим без звука
При включенной опции пользователю все сообщения от бота будут приходить без звука (в тихом режиме).
В опции "Приоритет истории сo звуком" можно задать уровень важности сообщений истории, которые будут приходить со звуком.
При отправке сообщений из своих скриптов/методов можно принудительно задать параметр отправки со звуком или без (см. ниже).

Команды
Команды используются для построения клавиатуры в клиенте. Одним нажатием кнопки из этой клавиатуры осуществляется какое либо действие, в зависимости от типа кнопки.
Тип кнопки
 Описание
textОтправляет сообщение со своим именем. Имеет поле Код. Также можно отлавливать в событии, как текстовое сообщение
locationИспользуется для отправки местоположения. Нажатие можно отловить в событии "Местоположение". Также, если отправитель привязан к пользователю системы, передаёт координаты в модуль GPS трекер
vkpayОткрывает окно оплаты VK Pay с предопределёнными параметрами. Параметры задаются в поле "Данные" в виде строки, содержащей параметры платежа VK Pay и идентификатор приложения в параметре aid, отделённый &. Пример: action=transfer-to-group&group_id=1&aid=10.
open_linkОткрывает ссылку, указанную в поле "Данные"
open_appОткрывает мини-приложение или игру, идентификатор которой указан в поле "Данные"
callbackНе отправляет сообщение, но отправляет факт нажатия на сервер. Имеет поле Код. События именно этого типа кнопок отлавливаются в событии callback. В данном случае $callback (или $payload) = Имя кнопки
Для типов text и callback можно выбрать цвет.
Цвет
 Описание
primaryИспользуется для кнопок, выполняющих основное действие.
secondaryИспользуется для обычных кнопок.
negativeИспользуется для кнопок, выполняющих опасное действие или отмену, например «Удалить» или «Отклонить»..
positiveИспользуется для кнопок, с помощью которых пользователь выражает согласие или подтверждает свои действия, например «Согласиться» или «Подтвердить».
У каждой кнопки есть настройка контроля доступа, где можно задать пользователей, которым данная команда доступна.
При нажатии на кнопку в клиенте выполняется код из поля код
Для отправки ответа на команду в конце кода должен быть return с текстовым сообщением.
Можно сформировать в коде клавиатуру с помощью функций buildKeyBoardButton() и buildKeyBoard(). Имя переменной клавиатуры должно быть $keyboard. В таком случае, клавиатура будет передана с сообщением return
В коде команды через переменную $user_id можно узнать ID пользователя нажавшего кнопку.
return "Ваш UserID - ".$user_id;
События
Данный функционал позволяет навесить на события, такие как получение текстового сообщения, файла, граффити, местоположения и др, свои обработчик.
Например, при получении местоположения, выполнить какие-то действия с координатами.
Для каждого из типов событий в выполняемый код передаются определенные параметры, такие как например $user_id.
Более подробно о параметрах расписано в подсказке, которая появляется при выборе типа события.
Для выхода из обработчика событий необходимо задать в коде обраюотчика параметр $skip=true;. Это нужно для того чтобы модуль перестал вызывать все подходящие обработчики, когда нужный сработал.
Событие callback используется для определения нажатия callback кнопок. Идентификация происходит по информации из параметра $payload функции buildKeyBoardButton(). Если $payload при создании кнопки задан не был, туда добавляется имя кнопки.
Формирование клавиатуры
Чтобы сформировать клавиатуру в коде, нужно действовать по следующему алгоритму:
1.Формируем кнопки клавиатуры с помощью функции buildKeyBoardButton($name, $type, $payload, $color, $data)
Параметр
 Описание
 Возможные значения
$nameИмя кнопкитекст
$typeТип кнопки, согласно описанию из раздела "Команды"text, location, vkpay, open_link, open_app, callback
$payloadДанные для определения, что была нажата конкретная кнопка в событии callback. Если пустой, чтановится равен имени кнопки.текст
$colorЦвет кнопки для типов text и callback согласно описанию из раздела "Команды"primary, secondary, positiv, negative
$dataДанные для кнопок vkpay, open_link, open_appПараметры платежа, ссылка или ID приложения
Параметры начиная с $payload необязательны для заполнения.
Формируем таким образом необходимое количество кнопок.
		$buttons[] = $this->buildKeyBoardButton('Кнопка 1', 'text', '', 'secondary);
		$buttons[] = $this->buildKeyBoardButton('Кнопка 2', 'location');
		$buttons[] = $this->buildKeyBoardButton('Кнопка 3', 'open_link, '', '', 'https://vk.com');
		$buttons[] = $this->buildKeyBoardButton('Кнопка 4', 'callback, 'button4', 'positive');
2. Формируем клавиатуру с помощью функции buildKeyBoard($buttons, $inline, $one_time)
Параметр
 Описание
 Возможные значения
$buttonsМассив, сформированный на этапе 1массив кнопок, сформированный buildKeyBoardButton()
$inlineТип клавиатуры, true - клавиатура в сообщении, false - под полем ввода текстаtrue, false
$one_timeЕсли клавиатура не inline, то данный параметр скрывает клавиатуру после нажатия на кнопкуtrue, false
Клавиатура готова! Можно передать её пользователю.
		$keyboard = $this->buildKeyBoard($buttons, true);
		$this->sendMessageToUser($user_id, 'Клавиатура для Вас', $keyboard);
Нужно иметь ввиду, что не-inline клавиатура будет заменена сформированной в модуле клавиатурой при отправке следующего сообщения.
Webhook

Для чего?
Для того, что бы исключить постоянный опрос серверов на наличие новых сообщений. Сервера Вконтакте сами уведомят о поступлении нового сообщения.

Как включить?
1. Ваш сервер должен быть доступен из интернет, а именно страница https://ваш_сервер/webhook_vkmessenger.php
2. В поле Webhook URL задается имя вашего сервера или ip
3. Нажатием кнопки "Set Webhook" отправляется запрос на установку вашего адреса для webhook. Результат отобразится ниже кнопки.

Плюсы
1. Быстрый ответ на команду
2. Нет постоянных запросов на наличие новых сообщений
3. Работает независимо от того, запущен основной цикл или нет.

Текущий статус использования webhook можно узнать нажав кнопку "Status webhook"

Вызов модуля в своих скриптах/методах

Для отправки текстовых сообщений
include_once(DIR_MODULES . 'vkmessenger/vkmessenger.class.php');
$vkmessenger_module = new vkmessenger();
$text = "test message";
$vkmessenger_module->sendMessageToUser($user, $text);
$vkmessenger_module->sendMessageToUser($user, $text, '', true); // без звука
$vkmessenger_module->sendMessageToAdmin($text." to admin");
$vkmessenger_module->sendMessageToAdmin($text, true);// без звука
$vkmessenger_module->sendMessageToAll($text." to all");
$vkmessenger_module->sendMessageToAll($text, true);// без звука

Для отправки изображений
include_once(DIR_MODULES . 'vkmessenger/vkmessenger.class.php');
$vkmessenger_module = new vkmessenger();
$img = '/var/www/images/trees.jpg';
$vkmessenger_module->sendImageToUser(139809588, $img, 'Описание');
$vkmessenger_module->sendImageToUser($user, $img, 'Описание', '', true); // без звука
$vkmessenger_module->sendImageToAdmin($img);
$vkmessenger_module->sendImageToAdmin($img, 'Описание','', true); // без звука
$vkmessenger_module->sendImageToAll($img);
$vkmessenger_module->sendImageToAll($img, '', '', true); // без звука
